This is a Particle Jekyll theme - a minimalist portfolio template for developers using Jekyll static site generator with Particle.js animations.

## Development Commands

### Setup
```bash

```

### Development
```bash

gulp
# or
npm run dev
```

### Jekyll Commands
```bash
# Build the Jekyll site
jekyll build

# The gulpfile handles Jekyll builds automatically during development
```

## Architecture

### Build System
- **Gulp** (v4.0.0) orchestrates all build tasks
- **Jekyll** generates the static site
- **BrowserSync** provides live reload during development
- Default task runs: js, sass, fonts compilation, then browser-sync with watch

### Directory Structure
- `src/` - Source files for assets
  - `styles/` - SCSS files compiled to `assets/css/`
  - `js/` - JavaScript files concatenated and minified to `assets/js/main.js`
  - `fonts/` - Font files copied to `assets/fonts/`
  - `img/` - Images optimized and copied to `assets/img/`
- `_includes/` - Jekyll partial templates
- `_layouts/` - Jekyll page layouts
- `_site/` - Generated static site output (git-ignored)
- `assets/` - Compiled/processed assets served to the site

### Key Files
- `_config.yml` - Jekyll and site configuration
- `gulpfile.js` - Build task definitions
- `index.html` - Main site entry point
- `package.json` - Node dependencies

### Gulp Tasks
- `jekyll-build` - Builds Jekyll site
- `jekyll-rebuild` - Rebuilds Jekyll and reloads browser
- `browser-sync` - Starts development server on `_site` directory
- `sass` - Compiles SCSS to CSS with minification
- `js` - Concatenates and minifies JavaScript
- `fonts` - Copies font files
- `imagemin` - Optimizes images
- `watch` - Monitors file changes and triggers rebuilds

### Configuration Notes
- Jekyll excludes: `package.json`, `src`, `node_modules` directories
- Site is served from `_site` directory during development
- Particle.js configuration is in the compiled JavaScript files

# CLAUDE.md

This file provides guidance to Claude Code when working with this repository.

## Project Overview

This is a **Particle Jekyll Theme** - a minimalist portfolio website template built with Jekyll static site generator. It features animated particle backgrounds using Particle.js.

## Tech Stack

- **Static Site Generator**: Jekyll
- **Build Tool**: Gulp 4
- **CSS Preprocessor**: SASS/SCSS
- **JavaScript**: Vanilla JS with Particle.js for animations
- **Package Manager**: Yarn
- **Icons**: Font Awesome and Devicon

## Project Structure

```
├── _config.yml          # Jekyll configuration and site settings
├── _includes/           # HTML partials (header, footer, about, projects)
├── _layouts/            # Page layout templates
├── assets/              # Compiled assets (generated by Gulp)
├── src/                 # Source files
│   ├── fonts/           # Icon fonts (Font Awesome, Devicon)
│   ├── img/             # Source images
│   ├── js/              # JavaScript source (app.js with Particle.js config)
│   └── styles/          # SCSS source files
│       ├── _vars.scss   # Color and style variables
│       └── main.scss    # Main stylesheet entry point
├── gulpfile.js          # Gulp build configuration
└── index.html           # Main page entry point
```

## Common Commands

```bash
# Install dependencies
yarn

# Run development server (builds Jekyll, compiles assets, starts BrowserSync)
gulp

# Alternative using npm script
yarn dev
```

## Build Process

The Gulp workflow handles:
- **SASS compilation** (`src/styles/` → `assets/css/`)
- **JavaScript minification** (`src/js/` → `assets/js/`)
- **Image optimization** (`src/img/` → `assets/img/`)
- **Font copying** (`src/fonts/` → `assets/fonts/`)
- **Jekyll build** (generates `_site/` directory)
- **BrowserSync** for live reload during development

## Customization Points

- **Site settings**: Edit `_config.yml` for user info, social links, descriptions
- **Colors/theme**: Edit `src/styles/_vars.scss` for color variables
- **Particle animation**: Edit particle configuration in `src/js/app.js`
- **Content sections**: Edit HTML files in `_includes/`

## Dependencies

Jekyll must be installed separately via Ruby gems:
```bash
gem install bundler jekyll
```

# Jekyll Portfolio Project Instructions

## Project Overview

This guide will help you set up a professional GitHub portfolio and freelancer work page using the **Particle** Jekyll theme. By the end, you'll have a live site at `rubenjansendata.com` showcasing your projects, skills, and freelance services.

**Theme Repository:** https://github.com/nrandecker/particle
**Your Domain:** rubenjansendata.com (via Namecheap)
**Development Environment:** Mac with Cursor

## Phase 1: Local Environment Setup

### 1.1 Install Prerequisites

Open your terminal and install the required tools:

```bash
# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Ruby (Jekyll requires Ruby)
brew install ruby

# Add Ruby to your PATH (add this to your ~/.zshrc or ~/.bash_profile)
echo 'export PATH="/opt/homebrew/opt/ruby/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Verify Ruby installation
ruby -v

# Install Jekyll and Bundler
gem install jekyll bundler
```

### 1.2 Clone the Particle Theme

```bash
# Navigate to where you want your project
cd ~/Projects  # or your preferred directory

# Clone the Particle theme repository
git clone https://github.com/nrandecker/particle.git rubenjansendata-portfolio

# Enter the project directory
cd rubenjansendata-portfolio
```

### 1.3 Install Dependencies and Test Locally

```bash
# Install project dependencies
bundle install

# Start the local development server
bundle exec jekyll serve
```

Open your browser and go to `http://localhost:4000` to see the default theme.

## Phase 2: Git Repository Setup

### 2.1 Create Your GitHub Repository

1. Go to https://github.com/new
2. Name the repository: `rubenjansendata.github.io` (this naming convention enables GitHub Pages)
3. Set visibility to **Public**
4. Do NOT initialize with README, .gitignore, or license (you already have files from the clone)
5. Click **Create repository**

### 2.2 Connect Your Local Project to GitHub

```bash
# Remove the original remote (points to nrandecker's repo)
git remote remove origin

# Add your new repository as the origin
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/rubenjansendata.github.io.git

# Verify the remote
git remote -v

# Push your code to GitHub
git branch -M main
git push -u origin main
```

### 2.3 Basic Git Workflow

Use these commands as you make changes:

```bash
# Check what files have changed
git status

# Stage all changes
git add .

# Commit with a descriptive message
git commit -m "Update: description of what you changed"

# Push to GitHub
git push
```

## Phase 3: Customize Your Portfolio Content

### 3.1 Main Configuration (_config.yml)

Open `_config.yml` in Cursor and update these key settings:

```yaml

title: Ruben Jansen | Data Professional
description: "Data analyst and freelancer specializing in [your specialties]"
url: "https://rubenjansendata.com"
baseurl: ""

# Social Links (update with your actual profiles)
linkedin: your-linkedin-username
github: your-github-username
twitter: your-twitter-handle  # optional
email: your@email.com

# Analytics (optional - add your Google Analytics ID)
google-analytics: UA-XXXXXXXXX-X
```

### 3.2 Personal Information

Edit the main page content. In Particle theme, look for these files:

**_config.yml** - Update the following sections:
```yaml
# About Section
about-title: About Me
about-description: >
  Write 2-3 paragraphs about yourself, your background,
  and what drives you professionally.

# Skills Section
skills-title: Skills
skills:
  - skill: Python
    value: 90
  - skill: Data Analysis
    value: 85
  - skill: SQL
    value: 80
  # Add more skills as needed
```

### 3.3 Projects Section

Create or edit project entries. In the Particle theme, projects are typically defined in `_config.yml` or in a `_projects` folder:

```yaml
# Projects Section
projects-title: Projects

projects:
  - title: Project Name
    description: Brief description of what the project does and the problem it solves
    image: assets/img/project1.png
    url: https://github.com/yourusername/project-repo

  - title: Another Project
    description: Another project description
    image: assets/img/project2.png
    url: https://link-to-project.com
```

### 3.4 Freelance Services Section

Add a services section to advertise your freelance work. You may need to modify the theme slightly. Create or edit content to include:

```yaml
# Services Section (you may need to add this)
services-title: Freelance Services
services-description: >
  Available for freelance projects in data analysis, visualization,
  and business intelligence.

services:
  - title: Data Analysis
    description: Transform raw data into actionable insights
    icon: fa-chart-bar

  - title: Dashboard Development
    description: Custom dashboards and reporting solutions
    icon: fa-tachometer-alt

  - title: Data Consulting
    description: Strategic advice on data infrastructure and processes
```

### 3.5 Add Your Profile Image

1. Add your professional photo to `assets/img/` directory
2. Name it something like `profile.jpg`
3. Update the reference in `_config.yml`:


## Phase 4: Enable GitHub Pages

### 4.1 Configure GitHub Pages

1. Go to your repository on GitHub
2. Click **Settings** → **Pages** (in the left sidebar)
3. Under "Source", select **Deploy from a branch**
4. Select **main** branch and **/ (root)** folder
5. Click **Save**

Your site will be live at `https://YOUR_USERNAME.github.io/rubenjansendata.github.io` within a few minutes.

## Phase 5: Custom Domain Setup

### 5.1 Add Domain to GitHub

1. In GitHub repository Settings → Pages
2. Under "Custom domain", enter: `rubenjansendata.com`
3. Click **Save**
4. Check "Enforce HTTPS" once DNS propagates (may take up to 24 hours)

### 5.2 Configure Namecheap DNS

1. Log in to Namecheap
2. Go to **Domain List** → Click **Manage** next to rubenjansendata.com
3. Go to **Advanced DNS** tab
4. Remove any existing A records or CNAME records for @ and www
5. Add the following records:

**A Records (for apex domain - rubenjansendata.com):**
| Type | Host | Value | TTL |
|------|------|-------|-----|
| A Record | @ | 185.199.108.153 | Automatic |
| A Record | @ | 185.199.109.153 | Automatic |
| A Record | @ | 185.199.110.153 | Automatic |
| A Record | @ | 185.199.111.153 | Automatic |

**CNAME Record (for www subdomain):**
| Type | Host | Value | TTL |
|------|------|-------|-----|
| CNAME | www | YOUR_USERNAME.github.io. | Automatic |

### 5.3 Create CNAME File

In your project root, create a file named `CNAME` (no extension) containing:

```
rubenjansendata.com
```

Commit and push this file:

```bash
echo "rubenjansendata.com" > CNAME
git add CNAME
git commit -m "Add custom domain CNAME"
git push
```

### 5.4 Verify DNS Propagation

DNS changes can take 24-48 hours to fully propagate. Check status:

```bash
# Check if DNS is resolving correctly
dig rubenjansendata.com +noall +answer

# Or use online tools like:
# https://dnschecker.org
```

## Troubleshooting

### Jekyll Build Errors

```bash
# If bundle install fails
gem install bundler
bundle update

# If jekyll serve fails
bundle exec jekyll serve --trace  # Shows detailed errors
```

### DNS Issues

- Wait 24-48 hours after DNS changes
- Clear browser cache
- Try incognito/private browsing mode
- Verify DNS settings exactly match the values above

### GitHub Pages Not Updating

- Check the **Actions** tab in your repository for build errors
- Ensure `_config.yml` has no syntax errors
- Verify the branch name matches what's configured in Pages settings
